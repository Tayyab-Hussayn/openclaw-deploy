# OpenClaw Deploy — Project Context

## What this is
A self-hosted OpenClaw deployment kit built for Tayyab to distribute to clients.
Clients clone this repo, run `./setup.sh`, and have a fully working OpenClaw instance
with Ollama cloud models — no manual onboarding required.

**GitHub:** https://github.com/Tayyab-Hussayn/openclaw-deploy

---

## Key separation: personal vs client

Tayyab's personal OpenClaw config lives at `~/.openclaw/` (never in this repo):
- `~/.openclaw/openclaw.json` — has his real gateway token + Brave API key
- `~/.openclaw/credentials/whatsapp/` — his WhatsApp session
- `~/.openclaw/identity/device.json` — his device private key

This repo ships a **clean template** with:
- WhatsApp disabled, empty `allowFrom`
- Brave plugin removed
- Gateway token = placeholder (replaced with random token by setup.sh)
- 9 Ollama cloud models pre-configured

**Runtime files (generated by setup.sh, not in git):**
- `.env` — absolute paths for this machine (Docker won't expand `~`)
- `workspace/` — live agent workspace mounted into container
- `info.sh` — prints dashboard URL + token

## Scripts

### setup.sh
Fully autonomous installer. Handles:
1. Internet check
2. curl install
3. git install
4. Docker install (apt/pacman/dnf/yum + Ubuntu derivative detection)
5. Docker Compose install (v2 plugin preferred, v1 fallback, binary fallback)
6. Docker daemon start (systemd → service → direct dockerd)
7. Docker group permissions
8. Port conflict checks (18789, 18790)
9. Config copy + token generation (python3 → openssl → /dev/urandom fallback)
10. Workspace setup
11. .env generation with **absolute paths** (critical — Docker won't expand `~`)
12. Ollama install + start (official installer → systemd → nohup background)
13. Container pull + start
14. Generates info.sh

**Key bug that was fixed:** Docker Compose does NOT expand `~` in .env volume paths.
Always write `$HOME`-expanded absolute paths to `.env`.

**Key bug that was fixed:** `tr -dc | head -c 48` with `set -euo pipefail` causes SIGPIPE.
Fixed with `( set +o pipefail; ... )` subshell.

### onboard.sh
Interactive command prompt bridging user to OpenClaw CLI.
Command → actual CLI command mapping:
- `connect whatsapp` → `channels login --verbose`
- `connect telegram` → `channels login --channel telegram --verbose`
- `models` → arrow-key menu → `configure`
- `configure` → `configure`
- `onboard` → `onboard`
- `status` → `status`
- `doctor` → `doctor`
- `tui` → `tui`
- `dashboard` → runs `./info.sh`

CLI is run via: `docker compose run --rm -it openclaw-cli [command]`

### info.sh (generated at runtime)
Prints dashboard URL + token. On WSL, also prints the WSL IP as fallback.
Regenerated every time setup.sh runs.

## Docker setup

```yaml
# docker-compose.yml key points:
ports:
  - "18789:18789"   # Gateway (dashboard + API)
  - "18790:18790"   # Bridge
extra_hosts:
  - "host.docker.internal:host-gateway"   # Lets container reach host Ollama
command: ["node", "dist/index.js", "gateway", "--bind", "lan", "--port", "18789"]
```

`--bind lan` means gateway listens on all interfaces (not just loopback).
This is what makes it accessible from WSL → Windows browser.

## WSL access from Windows

Docker binds to `0.0.0.0` so port is accessible from Windows.
- Try: `http://localhost:18789` (works on Windows 11 / recent Win 10)
- Fallback: `http://<WSL_IP>:18789` (get IP with `hostname -I | awk '{print $1}'`)
